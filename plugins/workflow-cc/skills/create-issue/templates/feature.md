# Feature テンプレート

新機能追加用。User Story 形式が適切。

---

## User Story

```markdown
## User Story

**[ペルソナ]として**、[達成したいこと]したい。
**なぜなら**[理由・価値]だからだ。
```

- ペルソナはリポジトリのドメインから採る (CLAUDE.md・既存 issue を参照。例: 一般利用者 / 管理者 / 運用担当 / システム)
- 悪い例: 「ユーザーとして、ボタンをタップしたい。なぜならボタンがあるからだ」← 目的も価値も不明確

---

## Acceptance Criteria

```markdown
## Acceptance Criteria

### 正常系
- [ ] Given: [前提条件] / When: [操作] / Then: [期待結果]

### 異常系
- [ ] Given: [エラー条件] / When: [操作] / Then: [エラー処理]

### 境界値
- [ ] Given: [境界条件] / When: [操作] / Then: [期待結果]
```

| 要素 | ポイント |
|------|---------|
| Given | 具体的な状態・データを記載 |
| When | 1 つのアクションに絞る |
| Then | 検証可能な結果 (「〇〇が表示される」「〇〇に遷移する」) |
