# Bug テンプレート

バグ修正用。User Story 形式は不適切。再現手順 + 期待/実際の動作を記載。

---

## バグ報告セクション

```markdown
## バグ概要

<!-- 何が起きているか、1-2 文で -->

---

## 再現手順

1. [画面/機能]を開く
2. [入力/操作]を行う
3. [不具合]が発生

### 再現環境

| 項目 | 内容 |
|-----|------|
| 環境 | 開発 / ステージング / 本番 |
| バージョン / ブランチ | |
| 発生頻度 | 常時 / 時々 / 特定条件 |

---

## 期待する動作

<!-- 本来どうあるべきか -->

---

## 実際の動作

<!-- 現在何が起きているか -->

### エラー情報

- エラーメッセージ:
- ログ (アプリログ / 監視ツール):
- スクリーンショット:
```

---

## Acceptance Criteria

```markdown
## Acceptance Criteria

### 修正確認
- [ ] Given: 再現手順を実行 / When: 修正後 / Then: 期待する動作になる

### 回帰確認
- [ ] Given: 正常なデータ / When: 通常操作 / Then: 既存機能に影響なし

### 類似ケース
- [ ] Given: 類似する条件 / When: 同様の操作 / Then: 同じ問題が発生しない (同一パターンの他箇所も確認)
```

## 優先度判断

| 優先度 | 条件 |
|--------|------|
| critical | 本番で主要機能が使えない・データ破損 |
| high | 本番で一部ユーザーに影響、回避策なし |
| medium | 回避策あり、または開発環境のみ |
| low | 軽微な表示崩れ、エッジケースのみ |
