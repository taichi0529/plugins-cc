# Refactor テンプレート

リファクタリング用。User Story 形式は不適切。目的 + 変更内容 + 機能への影響なしを明記。

---

## リファクタリングセクション

```markdown
## 目的

<!-- なぜリファクタリングが必要か -->

### 現状の問題

- [ ] [具体的な問題1]
- [ ] [具体的な問題2]

### 期待される改善

- [ ] [改善効果1]

---

## 変更内容

### 変更対象

| ファイル / モジュール | 変更内容 |
|---------------------|---------|
| `<path>` | [変更概要] |

### Before / After

<!-- 代表的な箇所のコード例 -->

---

## 機能への影響

**このリファクタリングは機能変更を含まない**

- [ ] 外部インターフェース (画面・API・出力) に変更なし
- [ ] 既存のテストがすべてパスする (テスト未導入箇所は手動確認)
- [ ] 利用者から見た動作は同一
```

---

## Acceptance Criteria

```markdown
## Acceptance Criteria

### 機能維持
- [ ] Given: リファクタリング後 / When: 主要機能を操作 / Then: 動作に変化なし

### 品質改善
- [ ] Given: リファクタリング前後 / When: [品質指標 (静的解析警告数など)] を計測 / Then: 改善されている

### 回帰確認
- [ ] Given: リファクタリング後 / When: リポジトリの検証ゲート (lint / 静的解析 / テスト) を実行 / Then: すべて pass
```

---

## 判断基準

| やるべき | やるべきでない |
|---------|---------------|
| 明確な改善目標がある | 「なんとなく綺麗にしたい」 |
| 影響範囲が限定的 | 全体に影響する変更を一度に |
| 機能開発の前準備として | 機能開発と同時に |
