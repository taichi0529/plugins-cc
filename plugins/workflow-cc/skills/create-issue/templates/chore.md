# Chore テンプレート

雑務用 (依存更新、設定変更、ドキュメント)。User Story 形式は不適切。タスクリスト形式。

---

## 基本形

```markdown
## 概要

<!-- 何をするか、なぜ必要か -->

---

## タスク

- [ ] [タスク1]
- [ ] [タスク2]

---

## 影響範囲

<!-- このタスクが影響する範囲 -->
```

---

## タイプ別の追加要素

### 依存関係更新

```markdown
## 概要

[パッケージ名]を[現バージョン]から[新バージョン]に更新する。

### 更新理由
- [ ] セキュリティパッチ / 新機能 / 非推奨 API の解消 / その他

## タスク
- [ ] マニフェスト・lock ファイルを更新
- [ ] Breaking Changes の確認・対応
- [ ] リポジトリの検証ゲート (lint / 静的解析 / テスト) がすべて pass
- [ ] 動作確認

## 参考
- リリースノート: [URL]
- マイグレーションガイド: [URL]
```

### 設定変更

```markdown
## 変更内容

| 項目 | Before | After |
|-----|--------|-------|
| [設定1] | [値] | [値] |

## 影響範囲
- 影響する環境: 開発 / ステージング / 本番
- 影響する機能:
```

---

## Acceptance Criteria

```markdown
## Acceptance Criteria

- [ ] Given: 変更前の状態 / When: タスクを完了 / Then: 期待する状態になる
- [ ] Given: 既存機能 / When: 変更後に実行 / Then: 影響なく動作する
```
