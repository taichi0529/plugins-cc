# Task テンプレート

実装タスク用。1 PR で完結する具体的な作業。親 Feature の子イシューとして作成。

---

## 構成

```markdown
## 親イシュー

Relates to #XX <!-- 親 Feature へのリンク -->

## 概要

[このタスクで実装する内容を 1-2 文で]

## 実装内容

- [ ] [具体的な実装項目1]
- [ ] [具体的な実装項目2]

## Blocked by

<!-- 依存する他のイシュー -->
- #XX (理由: ...)

## 技術情報

### 対象モジュール / 画面

<!-- 実コードのパスで記載 -->

### 実装方針

<!-- 使用する技術、パターン、参照すべき類似実装 -->

### 影響範囲

## テスト要件

- [ ] リポジトリの検証ゲート (lint / 静的解析 / テスト) がすべて pass
- [ ] [必要な動作確認]

## 参考

- 関連 Issue/PR:
```

---

## 運用ルール

- **1 Task = 1 PR** の原則を守る
- Task は親 Feature のサブイシューとして紐づける (手順は SKILL.md 参照)
- ブランチ命名: `<type>/<issue番号>-<短い説明>` (implement-issue skill の規則と揃える)
- PR は解決済みベースブランチへ。本文に `Closes #<Task番号>` / `Relates to #<親番号>`
- DoD はリポジトリ側の解決済み DoD を使う (無ければ省略)
